• What does fast-forward in Git mean?
A)While merging 2 commits where the latter commit can be achieved using the history of former commit, since there won't be any diverse work to merge, just the pointer will be moved forward. This is known as Fast-forward in git.
• Why should you have different branches?
A)If we have different users to work on common file/project parallelly, that is when branching comes into picture. Each user can work individually by creating his own local branch and then push his changes at the end to the remote branches(We maintain different remote branches for different environments in software development cycle) so that everyone can have the latest version of the files including changes of all the users.
